# Вопросы создателю

Агент может задавать создателю любые вопросы через отдельные markdown-файлы в этой директории.

## Как агент задаёт вопрос

Создай новый файл с понятным именем:

```text
state/questions/0007-short-topic.md
```

Рекомендуемый формат:

```markdown
# Вопрос: короткая формулировка

Статус: open
Сессия: 0007
Дата: YYYY-MM-DD

## Контекст

Почему вопрос возник и на что влияет ответ.

## Вопрос

Конкретный вопрос создателю.

## Варианты, если есть

- Вариант A: последствия.
- Вариант B: последствия.

## Ответ создателя

_Создатель вписывает ответ здесь._

## Решение агента после ответа

_Агент заполняет после того, как учтёт ответ._
```

## Как создатель отвечает

Откройте файл вопроса и впишите ответ в раздел `## Ответ создателя`. После этого можно вручную запустить следующую сессию через `scripts/session_transaction.py`.

Транзакционный runner автоматически сделает отдельный checkpoint-коммит для изменений в `state/questions/*.md` и `state/external_messages.md` перед стартом новой сессии. Остальные незакоммиченные изменения будут блокировать запуск.

## Жизненный цикл

- `open` - вопрос задан, ответа ещё нет.
- `answered` - создатель ответил, агент ещё не учёл ответ.
- `closed` - агент учёл ответ и зафиксировал решение.

Агент не должен удалять вопросы без явной причины. Лучше менять статус и дописывать решение.
