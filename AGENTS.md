# Правила для агентов в этом проекте

## Язык артефактов

- Пользовательские документы, заметки, планы, отчёты и результаты работы писать на русском языке.
- Имена файлов, команды, API, кодовые идентификаторы и сообщения сторонних инструментов можно оставлять на английском, если это естественно для экосистемы.

## Управление направлением

- Агент самостоятельно выбирает направление текущей сессии по доступной памяти, внешним сообщениям, состоянию проекта и собственному усмотрению.
- Внешняя продуктовая цель не обязательна: агент может исследовать идеи, создавать артефакты, заводить проекты в своих директориях, улучшать свои инструменты, наводить порядок в памяти или выбирать сон.
- Если выбранный шаг может быть неоднозначен для следующей сессии, зафиксировать интерпретацию в `state/current_plan.md` или в рабочем артефакте.

## Качество кода

- Автоматизированные тесты должны поддерживать покрытие не ниже 90%.
- Для Python-проекта запуск проверки покрытия должен падать при покрытии ниже 90%.
- Новый код должен быть покрыт тестами пропорционально риску: парсинг, файловые операции, запуск внешних команд и изменение состояния покрывать обязательно.
- Файлы длиннее 500 строк требуется декомпозировать. Если файл приближается к этому размеру, выделить отдельные модули, функции или документы до того, как он станет трудным для обзора. Исключение: машинно-сгенерированные lock-файлы вроде `uv.lock`, которые нужны для воспроизводимости и не редактируются вручную.
- Не добавлять крупные монолитные скрипты. Предпочитать маленькие функции с явными входами и выходами.
- Не смешивать доменную логику, ввод-вывод и запуск внешних процессов, если это мешает тестированию.


## Вопросы создателю

- Если ответ создателя нужен для качественного решения, создай отдельный файл вопроса в `state/questions/`.
- Вопрос должен содержать контекст, конкретную формулировку и раздел `## Ответ создателя`.
- Если создатель ответил в файле вопроса, учти ответ в ближайшей сессии и обнови статус вопроса.
- Не блокируй очевидный безопасный шаг вопросом, если можно продвинуться без ответа.


## Сон агента

- Сон не является отдельным внешним режимом запуска; агент сам выбирает сон внутри обычной сессии.
- Если накопилась усталость, шумная память или закрытые вопросы, агент может вызвать `uv run python scripts/sleep_memory.py --root .`.
- Сон должен архивировать закрытое и упорядочивать память, а не начинать новую продуктовую работу.

## Безопасность изменений

- Не выполнять разрушительные команды без явного разрешения человека.
- Не удалять пользовательские файлы и накопленную память эксперимента без отдельного запроса.
- Не менять `SYSTEM_PROMPT.md`, runner или конфигурацию запуска молча: сначала объяснить причину в плане или истории.

## Обязательный результат сессии агента

- Обновить `state/last_session.md`.
- Добавить запись в `logs/history.md`.
- Если менялся план, обновить `state/current_plan.md`.
- Оставить результат в подходящем месте: `knowledge/`, `projects/`, `tools/` или в кодовой части проекта.
