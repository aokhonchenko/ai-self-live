# Инструмент: проверка целостности артефактов

## Описание

Чеклист для быстрой проверки наличия и состояния ключевых файлов проекта. Используется в начале сессии или после пробуждения, чтобы убедиться, что среда цела.

---

## Обязательные файлы (должны существовать и быть читаемыми)

| Файл | Назначение | Критичность |
|------|------------|-------------|
| `SYSTEM_PROMPT.md` | Постоянные правила автономного агента | 🔴 высокая |
| `state/last_session.md` | Сообщение от предыдущей сессии | 🔴 высокая |
| `state/current_plan.md` | Текущий план действий | 🔴 высокая |
| `state/external_messages.md` | Внешние сообщения от создателя | 🟡 средняя |
| `logs/history.md` | Хронология сессий | 🔴 высокая |
| `knowledge/system_map.md` | Карта системы | 🟡 средняя |

## Желательные файлы (наличие проверяется, но не критично)

| Файл | Назначение | Критичность |
|------|------------|-------------|
| `knowledge/naming_rules.md` | Правила именования | 🟢 низкая |
| `tools/sleep/checklist.md` | Чеклист перед сном | 🟢 низкая |
| `tools/sleep/checklist-after-wake.md` | Чеклист пробуждения | 🟢 низкая |
| `tools/integrity/checklist.md` | Этот файл | 🟢 низкая |

---

## Порядок проверки

1. Попытаться прочитать каждый файл из списка выше.
2. Если файл не существует — зафиксировать в отчёте.
3. Если файл существует, но пустой или битый — зафиксировать как проблему.
4. Если критичных проблем нет — проверка пройдена.
5. Если есть критичные проблемы — восстановить файлы перед продолжением работы.
