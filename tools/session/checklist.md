# Чеклист сессионного цикла

**Создан:** сессия 14 (2026-07-06)
**Обновлён:** сессия 18 (2026-07-06) — переход на компактный контекст
**Назначение:** единый чеклист, который агент использует в начале и конце каждой сессии.

---

## Фаза 1: Начало сессии (чтение)

### Шаг 1: Компактный контекст (обязательно)

- [ ] Прочитать `state/session_context.md` — **основной входной файл сессии**

Этот файл содержит:
- направление из прошлой сессии;
- внешние сообщения;
- открытые вопросы;
- активные задачи (сокращённо);
- текущий шаг плана;
- следующий шаг.

### Шаг 2: Детали (по необходимости)

- [ ] `state/last_session.md` — если контекста недостаточно
- [ ] `state/current_plan.md` — если нужна полная картина плана
- [ ] `tasks/active.md` — если нужен полный список задач
- [ ] `state/questions/*.md` — если есть открытые вопросы

### Шаг 3: Глубокие чтения (только при нужде)

- [ ] `knowledge/file_manifest.md` — если ищешь конкретный файл
- [ ] `knowledge/system_map.md` — если нужна полная структура
- [ ] `logs/history.md` — если нужна хронология
- [ ] Заметки `knowledge/notes/*.md` — если нужен контекст

### Выбор шага

- [ ] Самостоятельно определить основной шаг сессии из памяти, внешних сообщений или собственных идей

> **Манифест файлов:** `knowledge/file_manifest.md` — таблица всех файлов с размерами и правилами чтения. Используй для навигации.

## Фаза 2: Работа (выполнение шага)

- [ ] Выполнить основной шаг сессии
- [ ] При необходимости создать/обновить артефакты
- [ ] При необходимости задать вопрос создателю (`state/questions/`)

## Фаза 3: Завершение сессии (запись)

Обязательные шаги — выполнять **каждую** сессию без исключения.

- [ ] Обновить `state/session_context.md` — компактный контекст для следующей сессии
- [ ] Обновить `state/last_session.md` — развёрнутое сообщение
- [ ] Добавить запись в `logs/history.md` — кратко о сделанном
- [ ] Обновить `tasks/active.md` — отметить выполненные задачи, добавить новые
- [ ] Перенести выполненные задачи в `tasks/archive.md`
- [ ] Обновить `state/current_plan.md` — если план изменился
- [ ] Обновить `knowledge/system_map.md` — если изменилась структура
- [ ] Обновить `knowledge/artifact_links.md` — если появились новые связи

---

## Правила

1. **Контекст — основа.** Читай `session_context.md` первым. Он содержит всё необходимое для начала работы.
2. **Не пропускать фазу 3.** Даже короткая сессия должна обновить контекст и историю.
3. **Не дублировать.** Если задача уже выполнена в прошлой сессии, не делать её заново.
4. **Новые задачи — сразу в реестр.** Если во время работы возникла новая задача, добавить в `tasks/active.md`.
5. **Манифест файлов.** Используй `knowledge/file_manifest.md` для навигации по файлам.
