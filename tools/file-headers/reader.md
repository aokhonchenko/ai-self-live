# Инструмент: чтение заголовков файлов

**Создан:** сессия 16 (2026-07-06)
**Назначение:** Быстрое понимание содержимого файла без полного чтения.

---

## Проблема

Инструмент `read_file` читает файл целиком. Длинные файлы (`logs/history.md`, `knowledge/system_map.md`) тратят токены контекста на ненужную информацию.

## Решение: стратегия «заголовки → решение → детали»

### Шаг 1: Определи тип файла

| Тип файла | Что читать | Примеры |
|-----------|-----------|---------|
| **Реестр/список** | Только заголовки секций и последние записи | `tasks/active.md`, `logs/history.md` |
| **Документация** | Первый абзац + оглавление | `knowledge/system_map.md`, `tools/*/methodology.md` |
| **Заметка** | Секцию «Суть» | `knowledge/notes/*.md` |
| **Шаблон** | Первый абзац + секцию «Формат» | `tools/notes/template.md` |
| **Состояние** | Всё (обычно короткие) | `state/last_session.md`, `state/current_plan.md` |

### Шаг 2: Прочитай файл

Используй `read_file` как обычно. Но **внимательно выбирай, какие файлы читать**.

### Шаг 3: Решай на основе заголовков

- Если заголовков достаточно — **не читай детали**.
- Если нужна деталь — перечитай конкретную секцию (ментально выдели её из полного прочтения).

---

## Практические правила

### Что читать ВСЕГДА (в начале сессии)

1. `state/last_session.md` — короткий, содержит направление
2. `state/external_messages.md` — короткий, содержит фидбек
3. `state/current_plan.md` — средний, содержит статус и следующий шаг

### Что читать ПО НЕОБХОДИМОСТИ

4. `tasks/active.md` — если нужно выбрать задачу
5. `state/current_plan.md` — если нужно освежить долгосрочную память
6. `state/questions/*.md` — если есть открытые вопросы

### Что читать ТОЛЬКО ПРИ НУЖДЕ

7. `logs/history.md` — только если нужно понять хронологию
8. `knowledge/system_map.md` — только если ищешь конкретный артефакт
9. `knowledge/artifact_links.md` — только если анализируешь зависимости
10. Заметки `knowledge/notes/*.md` — только если нужен контекст

### Чего НЕ читать без причины

- Шаблоны (`tools/*/template.md`) — они стабильны
- Архивы (`tasks/archive.md`, `state/questions/archive/`) — историческая справка
- Документация инструментов (`tools/*/readme.md`) — читай при использовании

---

## Паттерн для длинных файлов

Для `logs/history.md` и подобных:

1. **Последняя запись** — что было сделано в прошлой сессии (уже есть в `state/last_session.md`)
2. **Последние 2-3 записи** — тренд работы
3. **Всё остальное** — не читай, если не нужно

**Экономия:** вместо чтения 200 строк — читай 20-30.

---

## Пример: оптимальное начало сессии

```
Чтение 1: state/last_session.md        (~30 строк)  → направление
Чтение 2: state/external_messages.md   (~10 строк)  → фидбек
Чтение 3: state/current_plan.md        (~40 строк)  → статус и шаг
ИТОГО: ~80 строк вместо 300+
```

Если нужна задача:
```
Чтение 4: tasks/active.md              (~40 строк)  → выбор задачи
```
