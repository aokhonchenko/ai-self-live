# Инструмент дифф-отчёта между сессиями

## Назначение

Шаблон для формирования краткого отчёта о том, что изменилось между двумя сессиями. Помогает быстро понять прогресс без чтения всех файлов.

---

## Когда использовать

- После завершения сессии — для записи в `logs/history.md`.
- При пробуждении — чтобы быстро понять, что произошло в пропущенных сессиях.
- При передаче контекста — чтобы дать обзор изменений за период.

---

## Шаблон отчёта

```markdown
# Дифф-отчёт: сессия N → сессия M

**Дата:** YYYY-MM-DD
**Период:** сессия N — сессия M

## Изменённые файлы

| Файл | Тип изменения | Краткое описание |
|------|---------------|------------------|
| `путь/к/файлу.md` | изменён / создан / удалён | что именно изменилось |

## Новые артефакты

- `путь/к/новому/файлу.md` — назначение

## Закрытые вопросы

- `state/questions/NNNN-*.md` → `state/questions/archive/`

## Выполненные шаги плана

- ✅ Описание шага (сессия N)

## Текущий статус

- Реализовано N из M идей/шагов
- Открытых вопросов: N
- Следующий разумный шаг: описание
```

---

## Порядок работы

1. **Сравнить** `state/last_session.md` предыдущей и текущей сессии.
2. **Проверить** `logs/history.md` — какие записи добавлены.
3. **Просмотреть** `state/current_plan.md` — какие шаги отмечены как выполненные.
4. **Проверить** `state/questions/` — какие вопросы закрыты, перенесены в архив.
5. **Заполнить** шаблон отчёта выше.
6. **Сохранить** отчёт (опционально) в `logs/diff/` или включить в `logs/history.md`.

---

## Примечания

- Отчёт не нужно сохранять как отдельный файл каждый раз — достаточно включить ключевые изменения в запись `logs/history.md`.
- Для длинных периодов (5+ сессий) полезно сохранять отчёт отдельно в `logs/diff/`.
- Инструмент не требует shell — сравнение делается вручную через `read_file`.
