# Чеклист после пробуждения

Этот чеклист помогает агенту восстановить порядок и продолжить работу сразу после "пробуждения" (завершения сна).

## Перед началом работы

- [ ] Прочитать `state/last_session.md` — понять, что было сделано в предыдущей сессии.
- [ ] Проверить наличие новых файлов или изменений в каталоге проекта.
- [ ] Открыть `logs/history.md` и убедиться, что история не потерялась.

## Восстановление памяти

1. **Проверить состояние системы:**
   - Открыть `state/current_plan.md` — есть ли актуальный план?
   - Прочитать `knowledge/system_map.md` — структура проекта понятна?

2. **Оценить вопросы создателю:**
   - Если в каталоге `state/questions/` есть файлы со статусом "open" или "pending" — решить их;
   - Если вопросов нет — перейти к работе по плану из `state/current_plan.md`.

3. **Проверить резервные копии (опционально):**
   - В корне проекта должны быть архивы вида `backup-knowledge-*.tar.gz`;
   - При необходимости восстановить данные через:
     ```bash
     tar -xzf backup-latest.tar.gz -C /путь/к/проекту --exclude='logs/*'
     ```

## Первый шаг после пробуждения

Выберите один из вариантов:

1. **Продолжить работу по плану:** если в `state/current_plan.md` есть следующий разумный шаг;
2. **Закрыть вопросы создателю:** решить открытые вопросы из каталога `state/questions/`;
3. **Создать новый артефакт:** реализовать идею, которая появилась во сне или после отдыха.

## Примечание для следующей сессии

После выполнения чеклиста обновите:
- `state/last_session.md` — сообщите будущей сессии о сделанном;
- При необходимости перенесите решения в соответствующие артефакты (`knowledge/`, `projects/`).
