# src/ — исходный код агента

`src/` содержит исполняемую логику и инструменты агента.

## Структура

```text
src/
├── agent/                    # Логика работы с контекстом агента
├── tools/                    # Инструменты агента
│   ├── <tool_name>/          # Директория отдельного инструмента
│   │   ├── core.py           # Основная реализация, если инструмент крупный
│   │   ├── tool.py           # schema/passport/handle для runtime-реестра
│   │   └── __init__.py       # Совместимость старых импортов
│   ├── _runtime.py           # Общие guard/helper-функции для tool.py
│   └── README.md             # Контракт создания инструментов
└── session_runner.py
```

## Инструменты

Инструменты живут в `src/tools`, а не в `scripts`. `scripts/file_tools.py` является тонким адаптером для локального runner'а: он сканирует `src/tools/*/tool.py`, собирает OpenAI-compatible schema, runtime-паспорт и dispatch вызовов.

Для нового инструмента создавай директорию `src/tools/<tool_name>/` и добавляй туда `tool.py` с функциями:

- `schema()` - описание OpenAI function tool;
- `passport()` - строка паспорта для модели;
- `handle(root, arguments)` - обработчик вызова.

Если инструмент уже был плоским файлом, его реализация должна лежать в `core.py`, а `__init__.py` должен сохранять старый импортный контракт.

Подробный контракт и пример: `src/tools/README.md`.
